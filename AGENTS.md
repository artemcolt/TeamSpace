# Project Instructions

## UI Layout

- Do not build the app UI from large isolated container cards around whole screens, headers, sidebars, or setup areas.
- Prefer a continuous workspace surface: full-width bands, aligned sections, and integrated panels that feel like one application, not separate boxed cards.
- Use Telegram-like application structure for operational screens: adjacent panes that touch each other, shared background, thin separators, and no outer gutters between major areas.
- Do not wrap the app header, setup sidebar, setup content, or full screen sections in rounded bordered cards. These areas should be bands or panes, not floating containers.
- Use cards only for repeated items, dialogs, small tools, or genuinely framed records.
- Avoid excessive outer padding, oversized gaps, and tall empty framed areas. Keep operational screens dense, direct, and easy to scan.
