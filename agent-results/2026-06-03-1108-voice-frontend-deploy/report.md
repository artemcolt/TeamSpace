# Поддержка голосовой связи и обновления фронта

## Что было сделано
- Добавлена backend-модель голосовой связи с водителем.
- Добавлена поддержка исходящих вызовов диспетчера к водителю.
- Добавлена регистрация запросов на вызов от водителя.
- Добавлена обработка ответа диспетчера на запрос водителя автоматическим дозвоном.
- Добавлен журнал событий связи для голосовых вызовов, запросов водителя и текстовых сообщений.
- Добавлена поддержка результата вызова, длительности разговора и ссылки на аудиозапись.
- Добавлен справочник шаблонов формализованных сообщений S01-S10.
- Добавлено API для создания, изменения и удаления шаблонов сообщений.
- Добавлено API для отправки формализованного сообщения водителю.
- Добавлено API для получения состояния канала связи по транспортному средству.
- Добавлена проверка наличия SIM-номера транспортного средства перед инициацией вызова.
- Добавлено поле SIM-номера в карточку транспортного средства.
- Добавлена автоматизация обновления фронтенда на стенде через сборку статического релиза.
- Добавлена локальная инструкция для команды обновления фронтенда.

## Проверка
- Выполнена сборка backend: `./gradlew build`.
- Выполнен перезапуск локального backend.
- Проверена доступность API справочника парков через HTTP.
- Проверена доступность служебного API статуса АСФРТ через HTTP.
- Проверена выдача шаблонов формализованных сообщений через HTTP.
- Проверена выдача состояния канала связи по транспортному средству через HTTP.
- Проверена отправка формализованного сообщения и запись события в журнал связи.
- Выполнена сборка frontend: `npm run build`.
- Проверен синтаксис скрипта обновления фронтенда: `bash -n`.

## Риски и заметки
- Реальная интеграция с SIP, GSM-шлюзом или корпоративной АТС не реализована.
- Автоматическая запись MP3-разговоров реализована как поле журнала для ссылки на запись, без подключения сервиса записи.
- Для инициации звонка требуется заполненный SIM-номер транспортного средства.
- События связи хранятся в документном хранилище приложения; при росте объема журнала может потребоваться отдельная табличная схема хранения.
